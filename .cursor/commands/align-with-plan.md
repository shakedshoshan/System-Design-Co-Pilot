# Align with plan

Use this when starting a feature or reviewing progress.

1. Read `System_Design_CoPilot_Plan.md` sections that match the current task (workflow, agents, stack, outputs, security).
2. State what the spec requires vs what exists in the repo (bullet gaps).
3. Propose the smallest next steps in a sensible build order (e.g. API contracts and persistence before full agent graph, or scaffold + one vertical slice—pick what fits the task).
4. If the spec and code disagree, flag it and default to the spec unless the user says otherwise.
